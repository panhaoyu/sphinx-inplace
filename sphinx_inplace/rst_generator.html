sphinx_inplace.rst_generator

class RstGenerator(env: Optional[jinja2.environment.Environment] = None)[source]

根据 python 文件结构生成 rst 文件结构。

  1. 对于不存在的文件，直接生成；

  2. 对于已存在的文件，判断是否更新，如果更新则覆写，否则不改变，以免触发 sphinx 的更新机制；

  3. 对于多余的文件，删除。

本方法首先根据 python 文件结构生成 rst 文件结构， 然后再将其余文件进行拷贝。 因此，如果您本方法生成的 *.rst 不满意的话，可以采用与 python 文件同名但不同后缀的文件覆写自动生成的文件。

Examples：

package
  |- __init__.py
  |- __init__.rst

则 __init__.py 的内容会被弃用，而 __init__.rst 的内容会被拷贝过去。

Notes

本程序旨在对文档进行增量更新，目前的方案是整体删除然后再进行构建。 这是一个非常低效的方案，应当判断是否需要更新，如果需要更新的话再进行重新复制。

generate_python_package_files(tree: sphinx_inplace.file_tree.PythonFileTreeNode, dst_dir: str, pkg_name: str) → str[source]

基于一个 python package 生成相应的文件夹。

这个函数可以理解为是这个类的主函数， 它将递归对子包进行生成。

Parameters

  • tree – 源文件，是一个 Node 节点。

  • dst_dir – 生成的 rst 文件结构。

  • pkg_name – 用于写入模版中的 package_name 。

Returns

暂无返回值

Notes

本模块区别于 shpinx-apidoc 的将所有文件放在一个文件夹的方案， 采用了层级的方案进行 rst 文件结构的组织。 这样生成的内容更直观，且可以包含非 *.py 文件。

Returns

创建的文件夹路径。
